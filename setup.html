Supabase 설정 전까지는 각 팀원의 브라우저 로컬 저장소를 임시 저장소로 사용합니다. 같은 브라우저에서는 데이터가 유지되지만, 다른 기기나 다른 브라우저에서는 데이터가 보이지 않습니다. Supabase를 연결하면 모든 팀원이 같은 클라우드 데이터를 공유합니다.
필요한 도구 (모두 무료)

Supabase — 무료 클라우드 데이터베이스

PostgreSQL 기반 오픈소스 데이터베이스입니다.
무료 플랜: 데이터베이스 500MB + API 요청 월 200만 건 (팀 7명에게 충분)
신용카드 불필요, 구글 계정으로 즉시 가입 가능
👉 supabase.com

GitHub Pages — 무료 웹 호스팅

전세계 개발자가 사용하는 코드 저장소입니다.
GitHub Pages 기능으로 HTML 파일을 무료 웹사이트로 배포 가능
영구 무료, 신용카드 불필요, 구글 계정 연동 가능
👉 github.com

SQL Editor에서 테이블 생성

프로젝트 생성 완료 후:
1. 왼쪽 메뉴 → "SQL Editor" 클릭
2. 아래 SQL 전체를 복사해서 에디터에 붙여넣기

-- ====================================================== -- 비교과센터 대시보드 테이블 생성 SQL -- 이 SQL 전체를 복사해서 SQL Editor에 붙여넣고 Run 클릭 -- ====================================================== CREATE TABLE IF NOT EXISTS programs ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, department text, name text NOT NULL, description text, year integer, deleted boolean DEFAULT false, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE TABLE IF NOT EXISTS tasks ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, department text, program_id text, title text NOT NULL, description text, assignee_id text, assignee_name text, status text DEFAULT 'todo', priority text DEFAULT 'medium', year integer, month integer, quarter integer, start_date text, due_date text, note text, deleted boolean DEFAULT false, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE TABLE IF NOT EXISTS budget ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, department text, program_id text, title text NOT NULL, budget_item text, category text, budgeted_amount numeric DEFAULT 0, executed_amount numeric DEFAULT 0, description text, execution_date text, assignee_id text, assignee_name text, year integer, month integer, quarter integer, has_receipt boolean DEFAULT false, deleted boolean DEFAULT false, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE TABLE IF NOT EXISTS evaluations ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, department text, eval_type text, title text NOT NULL, organization text, year integer, eval_date text, due_date text, status text DEFAULT 'preparing', assignee_name text, description text, deleted boolean DEFAULT false, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); CREATE TABLE IF NOT EXISTS notices ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, department text, title text NOT NULL, content text, severity text DEFAULT 'medium', date text, author_id text, author_name text, is_confirmed boolean DEFAULT false, year integer, month integer, deleted boolean DEFAULT false, created_at timestamptz DEFAULT now(), updated_at timestamptz DEFAULT now() ); -- 공개 읽기/쓰기 허용 (내부 팀 전용 — URL 키로 접근 제한) ALTER TABLE programs DISABLE ROW LEVEL SECURITY; ALTER TABLE tasks DISABLE ROW LEVEL SECURITY; ALTER TABLE budget DISABLE ROW LEVEL SECURITY; ALTER TABLE evaluations DISABLE ROW LEVEL SECURITY; js/config.js 파일 수정

다운로드한 파일 중 js/config.js를 메모장(또는 텍스트 편집기)으로 열어
아래처럼 복사한 값을 붙여넣으세요:

// js/config.js — 이 두 줄만 수정하면 됩니다 const SUPABASE_URL = 'https://여기에-내-Project-URL-입력.supabase.co'; const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6... (anon public key 전체)';
이 파일 하나만 수정하면 모든 대시보드가 Supabase에 연결됩니다.
다른 파일은 전혀 건드릴 필요 없습니다.
파일을 다운로드하려면?

STEP 1 — 현재 데이터 내보내기
1

대시보드에서 데이터 JSON 다운로드

각 대시보드 오른쪽 상단의 엑셀 다운로드 버튼으로 데이터를 Excel로 저장하거나,
아래 버튼으로 현재 브라우저에 저장된 데이터를 JSON 파일로 다운로드하세요.

현재 브라우저 저장 데이터 내보내기
데이터가 없다고 나오나요?
현재 페이지(젠스파크)는 클라우드 API를 사용 중이므로 브라우저 localStorage에는 데이터가 없을 수 있습니다.
이 경우 ② 젠스파크 Table API 직접 내보내기 방법을 사용하세요.
2

젠스파크 Table API에서 직접 내보내기

젠스파크 구독이 유지되는 동안 브라우저 주소창에 아래 URL을 입력해 데이터를 JSON으로 저장하세요.
각 링크를 열어서 Ctrl+S(저장) 또는 우클릭 → 다른 이름으로 저장으로 JSON 파일 저장:

# 현재 젠스파크 프로젝트 URL + 아래 경로로 접속: tables/tasks?limit=1000 tables/budget?limit=1000 tables/evaluations?limit=1000 tables/notices?limit=1000 tables/programs?limit=1000
각 URL을 브라우저에서 열면 JSON 형태로 데이터가 표시됩니다.
그대로 저장하거나 내용을 복사해서 tasks.json 파일로 저장하세요.
STEP 2 — Supabase에 데이터 Import
3

Supabase에 데이터 업로드

Supabase 설정 완료 후, 저장한 JSON 데이터를 아래 방법으로 Import 하세요:

방법 A: Supabase 대시보드에서 직접 Import (간단)

1. Supabase 프로젝트 → Table Editor 클릭
2. 테이블 선택 (예: tasks)
3. 우측 상단 "Import data" 버튼 클릭
4. CSV 또는 JSON 파일 업로드
5. 나머지 테이블도 반복

방법 B: SQL Editor에서 INSERT (정확)

1. Supabase → SQL Editor 클릭
2. 아래 형식으로 데이터를 INSERT 쿼리로 변환하여 실행:

INSERT INTO tasks (department, title, status, priority, year, month, created_at) VALUES ('system', 'CIEAT 유지보수', 'in_progress', 'high', 2026, 3, now()), ('rc', 'RC 프로그램명', 'todo', 'medium', 2026, 3, now());
Excel로 저장한 경우: Excel → CSV로 저장 → Supabase Table Editor에서 Import 가능합니다.
